All in One - Improved productivity

Users were spending most of their time looking up information in different locations in QAD (3rd Party software), which resulted in about 5-7 tabs opened. To improve their productivity, I connected to the software using ODBC to extract all the data. Once I was able to pull the data, I used sql to get the desired information. The end user enters the order number and it will display all the information they need onto one page.

Picking Dashboard

The picking department can print pick sheets from the dashboard and use their Radio Frequency (RF) scanners to scan the bar codes without having to manually type any information. It will also track user performance such as total picks, time of completion etc. The sequence of items to be picked is also adjusted so users are not jumping across locations but more of a linear path for reduce picking time.
